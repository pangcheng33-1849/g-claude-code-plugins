# 媒体处理

当飞书文档引用图片或文件时，要把正文提取和二进制资源获取分开处理。

对图片和白板，当前推荐链路是：

1. 读取正文
2. 拉取 blocks，确认是否存在图片块
3. 如果是普通图片块，优先尝试单图片直下
4. 如果是白板块，优先尝试 `GET /open-apis/board/v1/whiteboards/{whiteboard_id}/download_as_image`
5. 直下失败时，再回退到导出 docx
6. 从导出的 docx 包里提取 `word/media/*`
7. 生成图片清单
8. 用图片理解工具读取本地图片，再和正文合并总结

如果直下和导出都失败：

1. 保留正文和 warning
2. 在结果中标记 `browser_capture_fallback_recommended=true`
3. 用浏览器打开文档
4. 对图片区域截图
5. 再用 `view_image` 读取截图

始终说明：

- 是否存在嵌入媒体
- 是否已经下载
- 获取它们还需要哪些标识符或 token
- 图片是否已经进入后续图片理解步骤
- 如果是白板，是否已经尝试过 board-v1 下载接口
- 如果失败，失败点是在文档访问、白板读取权限，还是导出权限

如果用户要求“完整导出”，就必须同时包含：

- 正文内容
- 媒体获取方案
- 图片理解结果
